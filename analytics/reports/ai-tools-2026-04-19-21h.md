## 注目ツール TOP3

### 1. markdown-vault-mcp（出典: GitHub / pvliesdonk）
- **概要**: FTS5全文検索＋セマンティック検索・フロントマター対応のMarkdownコレクション管理MCPサーバー。インクリメンタル再インデックス対応。
- **適用場面**: note/Ghost向けコンテンツのナレッジベース化。過去記事・アフィリエイト商品レビュー・ペルソナ設定をMarkdownで蓄積し、Claude生成時に関連コンテキストを自動注入。「似た記事を書かない」重複防止にも活用可能。
- **導入難易度**: 低
- **推奨**: 今すぐ試す

---

### 2. aleph-docs（出典: GitHub / albertoferrazzoli）
- **概要**: Markdown・画像・動画・PDF・音声を統合したマルチモーダルベクトルDB（pgvector）＋MCPサーバー。Ebbinghaus忘却曲線による記憶管理付き。Docker構成で運用コスト極小。
- **適用場面**: YouTube台本・Instagram画像説明・A8商品PDFカタログをまとめてベクトル化し、Claude Haikuによる低コスト検索補助RAGを構築。Bridge Serverと連携してn8nワークフローからクエリ可能。ただしpgvector運用の知識が必要。
- **導入難易度**: 中
- **推奨**: 様子見（★0・リリース直後のため安定性確認後）

---

### 3. mcpx-cli（出典: GitHub / khang2916102006）
- **概要**: 複数AI CLIツール向けMCPサーバー設定を1つの中央コンフィグファイルに統合・デプロイするCLIツール。
- **適用場面**: 既存のPlaywright MCP・Claude Code CLI・xurl CLIに加え今後追加するMCPサーバーを一元管理。PM2との相性も良く、n8n + Bridge Server構成での設定ドリフト防止に有効。
- **導入難易度**: 低
- **推奨**: 様子見（★0・ドキュメント充実度を要確認）

## 業界トレンド

RAGとMCPの「ハイブリッド実装」が実用フェーズに入りつつあり、同一データソースに対してリアルタイムMCPアクセスとベクトル検索RAGを組み合わせる設計が注目されている（Reddit r/ClaudeAI）。また、Docker MCP Toolkitへの評価が高まっており、MCP群のコンテナ管理による再現性確保がトレンドになりつつある。SNS自動化のようなマルチツール構成では、設定管理の複雑さがボトルネックになるため、中央集権的なコンフィグ管理ツールへの需要が顕在化している。
