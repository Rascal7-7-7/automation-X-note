## はじめに：このプロジェクトで何をしているのか

私たちは **AIを使ってSNS運用を自動化**しています。

X（Twitter）・note・Instagram・YouTube の投稿を、毎日ほぼ自動で行うシステムを構築しています。  
手動で毎日投稿・分析・コンテンツ作成をすると数時間かかる作業を、**AIと自動化ツールの組み合わせで数分に短縮**しています。

あなたにこの環境を構築してもらう理由は、**同じシステムを使って一緒に運用・改善していくため**です。

---

## システム全体の流れ

```
あなた（Telegram でスマホから指示するだけ）
        ↓
  OpenClaw（AI の受付窓口）
        ↓
  Bridge Server（各ツールへの橋渡し）
     ↙        ↘
 n8n              Claude AI
（自動スケジュール）  （コンテンツ生成）
        ↓
  X / note / Instagram / YouTube へ自動投稿
```

## 各ツールの役割と「なぜ必要か」

### 1. Claude Code（AI コーディングアシスタント）

**何をするツール？**  
ターミナル（黒い画面）で動く AI です。コードの修正・新機能の追加・バグの調査などを自然言語で依頼できます。

**なぜ必要？**  
このシステムの設定ファイルやコードを修正するとき、プログラミングの知識がなくても「〇〇を変えて」と話しかけるだけで対応してくれます。システムの改善・カスタマイズをする際に必須です。

**使い方のイメージ**
```
ターミナルで:
$ claude

Claude Code が起動したら:
「Instagram の投稿時間を12時から15時に変えて」
「X の投稿文をもう少しカジュアルなトーンにして」
「エラーが出ているので直して」

→ Claude が自動でファイルを編集してくれる
```

---

### 2. n8n（スケジュール自動実行エンジン）

**何をするツール？**  
「毎朝7時に〇〇を実行する」のようなスケジュール管理を、ブラウザ上の画面（GUI）で設定できるツールです。

**なぜ必要？**  
投稿の自動化はタイミングが重要です。毎日決まった時間に投稿・リサーチ・分析を実行するために使います。一度設定すれば、何もしなくても毎日動き続けます。

**使い方のイメージ**
```
ブラウザで http://localhost:5678 を開く
→ ワークフロー一覧が表示される
→ 各ワークフローの「Active」スイッチをONにするだけ
→ 以降は自動で毎日実行される
```

**設定済みのスケジュール一覧**

| 時刻 | 実行内容 |
|------|---------|
| 07:00 | X のトレンドをリサーチ |
| 08:00 | Instagram コンテンツ生成（2アカウント分） |
| 08:30 / 12:30 / 18:30 | X に投稿 |
| 09:00 | YouTube ショート台本生成 |
| 12:00 | Instagram に投稿（2アカウント分） |
| 12:00 / 18:00 | X のトレンド投稿にいいね |
| 22:00 | Instagram インサイト収集・エンゲージメント分析 |
| 23:30 | YouTube Analytics 収集 |

**n8n の操作手順（初回のみ）**

1. ターミナルで `npm run n8n` を実行
2. ブラウザで `http://localhost:5678` を開く
3. 初回はメールアドレスとパスワードを登録する
4. 左メニュー「Workflows」→「Import from File」
5. `n8n/workflows/` フォルダ内の `.json` ファイルを全てインポート
6. 各ワークフローを開き「Active」トグルをON

> **注意**: Instagram・YouTube のワークフローは API 設定が完了してから Active にしてください。

### 3. OpenClaw（AI エージェントゲートウェイ）

**何をするツール？**  
Telegram（スマホのチャットアプリ）から AI に指示を送るための「受付窓口」です。スマホから「今日のX投稿対応して」と送るだけで、AI が動き出します。

**なぜ必要？**  
外出中・移動中でも、スマホから一言送るだけでコンテンツ生成・投稿指示ができます。PCを開かなくてもシステムを操作できます。

**使い方のイメージ**
```
Telegram で Bot にメッセージを送る:
「今日のX投稿対応して」
「Instagram コンテンツ作って」
「今週の分析レポートを見せて」

→ AI が自動で処理して結果を返信してくれる
```

**OpenClaw の設定手順**

1. Telegram で `@BotFather` に `/newbot` を送って Bot を作成
2. 発行された **Bot Token** をメモ
3. 設定ファイルを作成:

```bash
mkdir -p ~/.openclaw
```

`~/.openclaw/openclaw.json` を以下の内容で作成:

```json
{
  "gateway": "telegram",
  "token": "YOUR_BOT_TOKEN",
  "allowFrom": ["YOUR_TELEGRAM_USER_ID"],
  "soulPath": "/path/to/automation/openclaw/SOUL.md"
}
```

> `YOUR_TELEGRAM_USER_ID` の確認方法: Telegram で `@userinfobot` に `/start` を送ると ID が表示されます

4. OpenClaw を起動:

```bash
openclaw start
```

5. Telegram で Bot に `/start` を送り、応答があれば設定完了

### 4. Bridge Server（APIの橋渡し役）

**何をするツール？**  
n8n や OpenClaw からの指示を、実際のスクリプト（プログラム）に伝える中継役です。

**なぜ必要？**  
n8n は HTTP リクエスト（URLへのアクセス）しか送れません。Bridge Server がそのリクエストを受け取り、適切なスクリプトを呼び出します。ユーザーが直接意識する必要はほぼありません。

**起動方法**

```bash
npm run bridge
# → http://localhost:3001 で起動
```

---

### 5. xurl（X への投稿・いいね CLIツール）

**何をするツール？**  
コマンド1行で X（Twitter）に投稿・いいねができるツールです。

**なぜ必要？**  
X の公式 API を使って、プログラムから自動投稿するために必要です。OpenClaw に同梱されているので、別途インストールは不要です。

**使い方**

```bash
# 投稿
xurl post "今日のAI活用Tips！..."

# いいね
xurl like 1234567890

# 自分のアカウント確認
xurl whoami
```

---

### 6. Playwright / Lightpanda（ブラウザ自動操作）

**何をするツール？**  
プログラムがブラウザを自動操作して、Web サイトの情報を収集するツールです。

**なぜ必要？**  
X のトレンド収集・YouTube のリサーチは、API を使うとお金がかかります。そのため、ブラウザを自動操作して無料で情報を集めています。

> Lightpanda は Playwright より 9倍高速・メモリ消費 1/16 の高性能版です。YouTube リサーチに使用しています。

## 環境構築の手順（概要）

詳細は [GUIDE.md](./GUIDE.md) のステップ1〜9を参照してください。

```
① Node.js・ffmpeg・Zellij などのシステムツールをインストール

⑧ OpenClaw を設定して Telegram Bot と接続
⑨ Playwright ブラウザ・Lightpanda をインストール
⑩ 動作確認（Bridge Server・各モジュールのテスト）
```

## よくある質問

**Q. プログラミングの知識は必要ですか？**  
A. 基本的には不要です。設定ファイルへの API キーのコピペ・ターミナルでのコマンド実行ができれば問題ありません。コードの修正が必要な場合は Claude Code に依頼してください。

**Q. どのくらいお金がかかりますか？**  
A. ほとんどのツールは無料です。費用がかかるのは Claude API のみで、月 $6〜15（約900〜2200円）程度です。

**Q. 毎日どのくらい操作が必要ですか？**  
A. 一度設定が完了すれば、自動実行されます。週1回 note 記事の最終確認（10分程度）と、月1回のコンテンツ計画更新が主な手動作業です。

**Q. エラーが出たらどうすればいいですか？**  
A. まず [GUIDE.md のトラブルシューティング](./GUIDE.md#トラブルシューティング) を確認してください。解決しない場合は Claude Code に「エラーが出た」と伝えてログを見せれば対応してくれます。
